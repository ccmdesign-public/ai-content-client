## Key Takeaways

Claude Code commands are powerful, underused tools that can transform your workflow from setup to multi-agent management. Key insights include:

- **Start with `/init` and `/doctor`** to configure your project correctly and diagnose issues before coding.

- **Use `/plan` before execution** to save tokens and ensure Claude's approach aligns with your goals.

- **Leverage `/mcp`, `/batch`, and `/loop`** for advanced workflows, connecting to external tools, parallelizing tasks, and automating repetitive checks.

- **Manage context and costs** with `/compact`, `/context`, `/fresh-model`, and `/cost` to maintain performance and control spending.

## Summary

The video presents a tiered guide to mastering Claude Code commands, designed to take users from beginner to power user. The presenter, Duncan Rogoff, organizes over 50 commands into 10 functional tiers based on utility and frequency of use.

### Essential Setup & Control (Tiers 1-2)

The foundation begins with **Tier 1 'non-negotiables'**: `/login` for authentication, `/doctor` to diagnose installation issues, and the critical `/init` command. `/init` creates the `quad.md` file, which provides Claude with persistent context about your project, tools, preferences, and business details. `/config` or `/settings` opens the UI for theme and model configuration, while `/help` lists all available commands.

**Tier 2 covers core session management**: Commands like `/clear`, `/reset`, and `/new` wipe conversation history for a fresh start. `/fresh-model` allows switching between Claude's models (Haiku, Sonnet, Opus) mid-session to optimize for cost and task complexity. `/cost` and `/usage` monitor token spending and rate limits. `/permissions` controls what tools and file types Claude can access, and `/compact` compresses conversation history without fully clearing it. `/plan` forces Claude into a planning mode to think through problems before acting, saving execution tokens.

### Building & Reviewing Code (Tier 3)

This tier focuses on the development cycle. `/diff` shows uncommitted changes, `/rewind` or `/checkpoint` rolls back the codebase, and `/fork` branches the conversation to explore different solutions. `/memory` edits the `quad.md` file to manage what Claude remembers across sessions. `/security-review` scans for vulnerabilities before shipping code, and `/simplify` spawns three parallel agents to review and optimize recently changed files. `/btw` allows for side questions without polluting the main conversation context, and `/debug` troubleshoots the current session.

### Power User & Automation (Tier 4)

Here, users delegate full missions to Claude. `/mcp` (Model Context Protocol) is highlighted as huge, enabling connections to third-party platforms like Notion, Slack, and Google Drive. `/batch` splits large jobs into parallel tasks, opening separate GitHub PRs for each. `/loop` runs a prompt repeatedly on a set interval, useful for monitoring deploys or PRs. Other commands manage plugins, multi-agent workflows (`/agents`), automated hooks, and the library of over 60,000 specialized skills.

### Workflow, Output & Environment (Tiers 5-10)

**Tier 5** handles output with commands like `/export`, `/copy`, and integrations for GitHub and Slack. **Tier 6** manages IDE and terminal environments. **Tiers 7-9** cover personalization (themes, colors), informational stats, and situational commands like remote control or gifting passes. **Tier 10** lists deprecated commands or novelties like `/stickers` to order Claude Code merchandise.
