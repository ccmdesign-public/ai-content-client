## Key Takeaways

This video demonstrates four practical workflows to overcome the generic, cookie-cutter designs produced by AI agents and build unique, beautiful websites with Google Stitch 2.0 and Claude Code. • **Leverage a Design System** using the `design.md` file to ensure visual consistency and transferable styling between agents and projects. • **Import and Adapt Existing Styles** by using screenshots or website URLs as style references, allowing Stitch to extract patterns without direct copying. • **Use Agent Skills for Automation** like the Enhanced Prompt, Stitch Loop, and React Component skills to create a seamless, one-shot build workflow. • **Convert to Interactive UI Libraries** by using the ShadCN UI skill to transform static Stitch designs into interactive, component-based React applications.

## Summary

The core problem addressed is that AI agents, while proficient at generating functional code, often produce visually generic and similar-looking websites due to being trained on the same datasets. Google Stitch 2.0 tackles this by introducing a structured **design system** centered around a `design.md` file. This file, optimized for AI agents, defines colors, fonts, and themes, and is automatically created with every new Stitch project to maintain consistency.

### Method 1: Creating and Using Design Systems

You can create a custom `design.md` file or upload an existing one. Stitch visualizes this system, allowing you to preview the styling. This file can be generated by other agents (like Claude) using a template from Google's official skills repository, ensuring the structure is agent-optimized. Once saved, Stitch uses this system as the base for all subsequent pages, guaranteeing design consistency.

### Method 2: Importing Styles from Existing Websites

Instead of designing from scratch, you can provide Stitch with a style reference. The **Redesign feature** uses a full-page screenshot not to copy a site, but to extract its design language, patterns, and component placement as a style guide. You can also directly import a design system from any website by providing its URL, or upload a wireframe/sketch to specify your desired theme.

### Method 3: Automating with Agent Skills

Google provides specialized **skills** that integrate Stitch into agent workflows like Claude Code. Key skills include:

*   **Enhanced Prompt Skill:** Converts vague user prompts into Stitch-optimized prompts.

*   **Stitch Loop Skill:** Enables autonomous, iterative website building using Chrome DevTools.

*   **React Component Skill:** Converts Stitch's monolithic HTML output into modular React components for easier integration.
A recommended workflow involves creating a `claude.md` file that sequences these skills: first enhancing the prompt, then building the design via the Stitch Loop, and finally implementing it with React components.

### Method 4: Building with UI Component Libraries

To move beyond static designs, you can use the **ShadCN UI skill**. This skill provides guidance for converting Stitch designs into ShadCN components, which come with built-in interactions and animations. You can specify additional component registries (like Glassmorphism or Motion Primitives) to extend the library's capabilities. This workflow, powered by the ShadCN MCP, allows you to generate fully interactive applications that match your Stitch design in a single pass.
