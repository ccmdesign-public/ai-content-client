## Key Takeaways

AI engineer Karpati shares five actionable rules for scaling AI agent workflows based on running up to eight agents simultaneously. • **Remove human bottlenecks** to enable fully autonomous agent operation with minimal token input. • **Treat instructions as tunable code**—iterate on markdown files and let agents write better versions. • **Replace custom apps with agent-driven API glue**, unifying systems (like smart homes) through natural language interfaces.

## Summary

The video presents five rules from an AI engineer (Karpati) for designing effective AI agent systems, derived from his experience running up to eight agents concurrently.

**Rule 1: Remove Yourself as the Bottleneck**
The core principle is to architect workflows where AI agents run completely autonomously. The goal is to minimize human intervention—you should only need to provide "very few tokens occasionally" and then let "a huge amount of stuff happens on your behalf." This shifts the human role from constant prompter to system architect.

**Rule 2: Treat Agent Instructions as Tunable Code**
Agent instructions (often in markdown files) should not be treated as static documentation. Instead, view them as **iterative, tunable code**. Different instructions produce different behaviors. The recommended practice is to run variants, see which ones work, and even **let the agents themselves write better instructions** based on the results, creating a self-improving loop.

**Rule 3: Replace Bespoke Apps with Agent-Driven API Glue**
Instead of building or using multiple specialized applications, use agents as the unifying layer. The example given is "Carpassy" (likely a reference to Karpati) who unified his entire smart home—lights, HVAC, cameras, pool—into a single **WhatsApp assistant called Dobby**, controlled entirely through natural language. He argues the individual apps "shouldn't even exist."

**Rule 4: Invest in Persistent Looping Agent Setups**
Move beyond single-session, task-specific interactions. Set up agents that **keep running on your behalf** in a persistent state, monitoring and acting even when you're not actively engaged. This enables continuous, background automation.

**Rule 5: Focus Energy Exclusively on What Agents Can't Do**
Human value lies in **irreducible creative insight**, taste, and judgment. Your energy should be focused exclusively on these areas. Everything else that can be automated should be handed off to the agent system. The video ends by teasing that "part four has a study that says all of this might be wrong," suggesting a critical examination of these principles.
