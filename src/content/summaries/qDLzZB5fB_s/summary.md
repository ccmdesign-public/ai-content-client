### How Claude Code Works

Claude Code is an AI agent that operates within your local file system, with the ability to read documents, write files, execute commands, and connect to external tools via **Model Context Protocol (MCP)**. It maintains **persistent memory** through the claude.md file, which acts as a business brain loaded every session, containing rules, SOPs, brand voice, and data structures. This addresses three fundamental flaws of traditional AI: **context window amnesia** (forgetting information), **siloed sessions** (lack of cross-department communication), and **no operational execution** (inability to do real work like updating CRMs).

Key components include **skills** (packaged instructions in markdown files for tasks like proposal generation), **MCPs** (connecting to over 8,000 tools like Gmail, Slack, and Salesforce), and **sub-agents** (parallel AI instances for focused subtasks). For example, one sub-agent can research competitors while another drafts proposals, all running simultaneously without losing context. This architecture multiplies output, turning an 8-hour task into 30 minutes of supervised automation.

### Setting Up Claude Code

To get started, users need to download a free IDE such as **VS Code**, **Cursor**, **Anti-gravity**, or **Windsurf**—all built on VS Code foundations. Then, install the Claude Code extension from the IDE's marketplace, ensuring it has high ratings. Log in with an Anthropic subscription (starting at $20/month) or API key, with Nick recommending subscriptions over API for cost control. The first step is creating a claude.md file by typing "/init" in the IDE, which autogenerates a stub based on existing files.

Users should customize the claude.md file with company overview, key processes, brand voice, data structure, and tool connections, but start simple with one process to automate. The file is stored locally and can be shared via repositories like **GitHub** for team access. Nick demonstrates navigating folders in the IDE and adjusting layouts to view agents and files seamlessly, emphasizing that no technical prowess is required.

### Building Skills and Practical Applications

Skills are markdown files stored in a Claude skills folder, containing structured instructions with names, descriptions, tools needed, and step-by-step guides. Nick shows creating a **proposal generation skill** by pasting SOP content from a document into Claude Code, which then formats it into YAML for repeatable use. He enhances it to pull meeting transcripts from **Fireflies** and auto-generate proposals with contact details, deliverables, and investment summaries.

Examples of applications include **content repurposing** (turning blog posts into LinkedIn threads, emails, and YouTube scripts), **lead scraping** using **Apollo** and **Appify** for API calls, **email outreach automation**, and **operations** like CRM updates. Skills can be iterated by telling Claude what adjustments to make, and they stack—e.g., a lead qualification skill can trigger proposal generation. Nick notes that if a task can be done digitally, Claude Code can probably automate it, with the biggest lifts seen in marketing and content generation.
