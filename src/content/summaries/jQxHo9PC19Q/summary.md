## Key Takeaways

The video demonstrates a workflow for creating visually distinct, animated website hero sections to avoid generic AI-generated designs. • **Use Nano Banana Pro** for image generation with prompt templates and reference images for better results. • **Create subtle motion videos** (5-15 seconds) from images using Clling 3.0 or VO 3.1, prioritizing tasteful, slow animation. • **Build the page with Claude Code** by providing the video, a static image for mobile, and design inspiration (e.g., from 21st). • **Deploy for free** by pushing code to GitHub and hosting on Versel.

## Summary

The tutorial begins by emphasizing the goal of defeating the 'AI slot monster'—the generic, bland designs common in AI-generated websites—by adding animation to create a 'wow factor'. The presenter advocates for techniques that are both simple and high-leverage.

The workflow is broken into four main steps. First, create a starting image inside **Nano Banana Pro**. The presenter stresses the importance of both the prompt and inspiration. A provided prompt template helps structure ideas (camera type, lighting, atmosphere). For inspiration, sources like Pinterest or MidJourney can provide art styles, and reference images can be uploaded directly to guide the output. Landing page inspiration from sites like Dribbble or Godly helps define the desired visual composition and use of space. Iteration is expected to get the desired image.

Second, turn that image into a video using tools like **Clling** or **VO**. The prompt should be simple, instructing 'keep it static' with 'extremely slow and subtle motion'. The presenter notes that **Clling 3.0** and **VO 3.1** are solid options that accept a starting image. Two video length strategies are discussed: a longer 15-second video or a shorter 5-second one meant to be looped. A longer video offers more creative freedom and is simpler to implement, while a short loop can be created using techniques like reversing the second half with ffmpeg. The key advice is 'less is more'; animation should accentuate the page, not overwhelm it.

Third, use **Claude Code** to build the landing page. Create a new folder for the project, open Claude Code in it, and drop in the generated video and image. The core prompt instructs Claude Code to create a landing page for the AI agency 'Chase AI', using the video for an animated hero section and the static image for mobile users, employing its front-end design tool. Additional skills like **UIUX Pro** or **UIUX Pro Max** can be used for more refined design. Providing examples of websites to mimic (via screenshots) reduces iteration and improves outcomes. Resources like **21st** provide pre-built component prompts (e.g., for buttons) that can be copied into Claude Code for faster development. Once satisfied, a dev server can be spun up to preview the site.

Finally, deploy the site. Push the code to a GitHub repository (a free account is needed). Then, use **Versel** to deploy from that repository. Versel offers a generous free tier and integrates easily with GitHub. After deployment, the site is live with a URL that can later be customized.

## Context

As AI tools make website creation faster and more accessible, a common pitfall is ending up with generic, uninspired designs that lack visual impact. This tutorial provides a practical solution for developers, designers, and entrepreneurs who want to leverage AI for speed but also inject unique, engaging visual elements into their projects. It connects to broader trends of using multimodal AI (image, video, code generation) in a cohesive workflow to produce professional, animated web components without deep technical expertise.
