## Summary

The creator shares a highly efficient AI coding workflow centered around a single custom skill called **PR creator with docs**. This skill operates during the pull request creation phase, automatically analyzing all code changes made on a particular branch.

When triggered, the AI skill scans for relevant documentation files—specifically `.md` files—that need updating based on the recent code modifications. The creator maintains a systematic documentation structure where each major feature in their application has its own dedicated `.md` file, serving as a **source of truth** for that specific functionality.

This approach ensures that whenever code changes are made, the corresponding documentation is automatically synchronized, eliminating manual documentation updates that typically consume significant time. The system intelligently identifies which `.md` file corresponds to which feature update, maintaining consistency across the entire application.

Beyond this core skill, the creator mentions using **parallel AI agents** to work on multiple features simultaneously, creating a highly efficient development environment where different aspects of a project can progress concurrently without losing documentation integrity.

The workflow also includes systems for helping AI create better designs when implementing changes, suggesting a comprehensive approach to AI-assisted development that extends beyond just code generation to include design and documentation maintenance.
