## Key Takeaways

Designers are evolving their workflow by integrating AI coding assistants directly into their process, which changes prototyping, collaboration, and skill requirements. Key insights include using VS Code's **Plan Mode** for complex flows, creating separate **'vibe' environments** for safe prototyping, and seeing the future role shift towards being **judges and evaluators** of AI-generated UI.

## Summary

In a conversation between VS Code designer Eli and Microsoft Foundry designer Tua, they explore how designers are adapting to the rapid evolution of AI tools in their workflow. Tua shares her practical approach to navigating the constant stream of new AI models, agents, and terms by selectively committing to specific tools and testing them through small, practical projects.

### AI-Assisted Design Workflow

Tua details her use of **GitHub Copilot CLI** in the terminal and **VS Code** with its chat panel and agents. For complex user flows, she starts by visually mapping the experience in **Fig Jam**, then uses **VS Code's Plan Mode** to break it down step-by-step with AI assistance. She avoids 'oneshotting' complex prompts, instead working through steps in new chat sessions to maintain clarity. For simpler features, she directly prompts the agent with a Figma link via **MCP** to implement components.

### Prototyping in Code vs. Figma

Eli describes using VS Code's redesigned agent sessions to run multiple parallel background sessions in separate work trees. This allows for rapid A/B testing of variants (like button colors) directly in the codebase without affecting the main branch. This mirrors the iterative brainstorming process of Figma but with the crucial advantage of seeing changes **permeate across the entire product** via the live design system, revealing edge cases static prototypes miss.

### Collaboration and Role Evolution

Both designers discuss the collapsing boundaries between design, engineering, and product management. Tua uses **Vibe Foundry**, a designer-specific replica of the Microsoft Foundry front-end, as a playground to build and share exact front-end code with engineers. Eli highlights the advantage on the VS Code team where designers can clone the repo, make changes, and submit PRs, drastically shortening the distance from idea to public product.

The conversation concludes by examining the future of **GenUI** (generative UI), where agents create interfaces on-demand based on user prompts. This shifts the designer's role from creator to **judge, evaluator, and critic**, focusing on ensuring the AI's output aligns with design principles and user needs. The core challenge becomes defining the designer's place in a process where the agent and user prompts primarily drive the experience.

## Context

The rapid advancement of AI coding assistants is fundamentally changing software design workflows. This discussion is crucial for designers, product managers, and engineers navigating this shift. It addresses the pressure to adopt new tools, the evolution of prototyping from static mockups to live code, and the impending convergence of design and engineering roles. Understanding these changes is key to staying relevant and collaborative in modern product development.
