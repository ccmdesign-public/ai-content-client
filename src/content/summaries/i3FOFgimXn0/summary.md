## Summary

**Antigravity**, Google's free agent-first AI IDE for building autonomous agents, has received a major update with **AgentKit 2.0**. This update centers on a new **agent skill system** that allows agents to read structured instructions from `agent.md` files (in addition to `gemini.md`), enabling dynamic loading of modular behaviors. This unlocks a new level of customization, controlled output, and higher quality generations.

The **AgentKit** is a collection of templates, skills, and workflows designed to supercharge agent capabilities. It includes 16 specialized agents for domains like frontend, backend, and security, 40+ domain-specific knowledge skills, and 11 slash commands (like `/brainstorm`, `/create`, `/deploy`) for common development tasks. The kit automatically detects and applies the right specialists, using a spec-driven workflow to prevent hallucination and ensure high-quality outputs.

**Inforge** is a backend platform specifically designed for AI coding agents. It allows agents to automatically create and configure backends—including authentication, database schemas, cloud storage, and deployment—through simple prompts. The platform connects to Antigravity via a **CLI installation** and **MCP (Model Context Protocol)**, creating a visual canvas for backend communication. This integration means the Antigravity agent can handle all infrastructure setup autonomously and in record time.

The video demonstrates building a **full-stack finance tracker application (PayPulse)** from idea to live deployment in under an hour. The process starts with brainstorming using the AgentKit's `/brainstorm` command to create a spec. Then, prompts are sent to set up both the backend (via Inforge MCP) and frontend. The agent autonomously creates: a working authentication system (with GitHub OAuth), a database schema, a dashboard with expense tracking, receipt upload with OCR processing, budget management, analytics, and a polished landing page.

Finally, the app is deployed directly from Antigravity to Inforge with a simple command, resulting in a live, fully functional application. The workflow also showcases adding a **model gateway** to integrate LLM features (like using Gemini 3 Flash for receipt transcription) directly into the backend. The entire process highlights a shift toward AI-first development, where agents handle both code generation and infrastructure configuration seamlessly.
