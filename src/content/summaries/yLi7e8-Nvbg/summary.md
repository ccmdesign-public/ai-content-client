## Key Takeaways

The /loop feature in Claude Code enables persistent, automated workflows by scheduling recurring AI tasks. Key insights include: • **Session-based automation** that runs prompts repeatedly (every 10 minutes by default) while your terminal is open, up to 3 days • **Natural language scheduling** using intervals like minutes, hours, or days, and the ability to chain other Claude commands • **Practical use cases** like monitoring PRs, scanning logs for errors, and generating daily summaries, turning Claude into a background assistant • **Comparison to desktop app** where /loop is lightweight and session-based vs. the desktop's persistent scheduled tasks

## Summary

Claude Code has introduced a significant upgrade with the **/loop feature**, which fundamentally changes how developers interact with the AI coding assistant. This new capability allows users to schedule recurring tasks that run automatically at specified intervals, effectively turning Claude from a manually prompted tool into a **lightweight AI agent** that works continuously in the background.

The feature operates on a **session-based model**, meaning tasks will run repeatedly as long as your Claude Code terminal session remains open, with a maximum duration of **3 days** to prevent forgotten loops from consuming tokens indefinitely. The system is designed to be simple to use—you can initiate a loop with natural language commands like `/loop every 15 minutes, scan logs for new errors, and summarize what was found`. The interval syntax supports minutes, seconds, hours, or days, with a default of every 10 minutes if no interval is specified.

### Practical Applications

Developers can use /loop for various automation scenarios:

- **Continuous monitoring** of pull requests, build statuses, or deployments

- **Error detection and reporting** by scanning application logs at regular intervals

- **Research automation** for tracking AI model releases or industry news

- **Multi-agent workflows** where one agent identifies issues and deploys sub-agents to fix them

### Technical Implementation

To get started, users need to update their Claude Code instance and access the new /loop command. The feature supports up to **50 scheduled tasks per session** and includes management capabilities to list or cancel tasks at any time. Importantly, /loop can chain other Claude commands, making it possible to automate existing workflows without significant modification.

### Comparison with Desktop App

While similar to the scheduled tasks feature in the Claude Code Desktop app, the terminal-based /loop differs in key ways:

- **Session persistence**: /loop tasks expire when the terminal closes, while desktop tasks survive app restarts

- **Interface**: Desktop offers a better UI with clickable hyperlinks, while /loop is terminal-focused

- **Use case**: /loop is ideal for quick, session-based automation; desktop is better for persistent, long-running tasks

The feature represents a shift toward **agentic AI workflows** where developers can delegate repetitive monitoring and reporting tasks, freeing up mental bandwidth for more complex problem-solving while maintaining oversight through automated summaries and reports.

## Context

This update matters because it represents a significant evolution in AI-assisted development tools, moving from reactive assistance to proactive automation. As coding workflows become increasingly complex with continuous integration, deployment pipelines, and large codebases, developers need tools that can handle repetitive monitoring and reporting tasks autonomously. The /loop feature addresses this by transforming Claude Code from a coding assistant into a lightweight agent that works alongside developers 24/7. This aligns with broader trends in AI agent development and automation-first tooling, where AI systems are increasingly expected to execute tasks independently rather than just provide suggestions.
