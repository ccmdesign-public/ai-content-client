## Key Takeaways

The creator tested five different security audit prompts on a Laravel project generated by Claude Code.

- **Vague prompts can outperform specific ones:** A simple prompt from a Twitter thread found a critical stored XSS vulnerability that a detailed, custom Laravel audit prompt missed.

- **Results are non-deterministic:** Running the same security prompt multiple times yielded different findings, highlighting the inconsistent nature of current AI-powered audits.

- **Run audits multiple times:** For better coverage, it's advised to execute AI security checks several times, as each run may uncover different potential issues.

## Summary

The video documents an experiment testing five different AI-powered security audit prompts and skills within Claude Code on a Laravel project. The project was a fully functional accountant booking system generated by an AI agent, providing a real but unfamiliar codebase to audit.

The creator first used a custom, detailed **Laravel security audit** command they built. This prompt specifically instructed the AI to scan for framework-specific issues like CSRF protection in Blade templates and Eloquent model security. It returned a report with three medium and three low-priority findings, such as missing rate limiting and unencrypted sessions, in under three minutes.

### Testing the "Vague" Prompt

Next, the creator tested the original, more general prompt shared by Arvid Khal on Twitter. Despite its lack of specific instructions, the Claude Code agent intelligently spawned sub-agents to parallelize scanning for common vulnerabilities like SQL injection and XSS. This scan identified a **critical stored XSS vulnerability** in a Blade template where user-controlled JSON data was rendered unsafely—a finding the specialized Laravel audit missed.

### The Non-Determinism Problem

A key revelation was the **non-deterministic nature** of these AI audits. When the same 'auto research security' prompt was run a second time, it automatically triggered the creator's Laravel skill but produced a different report, finding only one medium and one low issue instead of the original six. This demonstrates that AI security checks are not reliable for consistent, deterministic scanning.

The creator concludes that while these free, AI-powered tools are "better than nothing" and can uncover serious issues, they should not replace dedicated, deterministic security scanners for specific languages or frameworks. The best practice is to run them multiple times to increase the chance of finding diverse vulnerabilities.

## Context

As AI-assisted coding becomes more prevalent, developers are increasingly relying on AI agents to generate and review code. However, the security of AI-generated codebases is a major concern. This video explores whether current AI tools can effectively audit their own output for security vulnerabilities, a critical skill for ensuring safe applications in an AI-driven development workflow. This matters to developers, DevOps engineers, and security professionals who integrate AI coding assistants into their pipelines and need to validate the security of the resulting code.
