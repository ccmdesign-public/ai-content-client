* Use **Nano Banana 2** and **Cling 3.0** via **key.ai** to create product animation videos from simple image prompts.

* Leverage **Claude Code skills** (video-to-website & front-end design) to automatically build scroll-driven animated websites from video inputs.

* Implement a professional **deployment pipeline** using **GitHub** for version control and **Vercel** for automatic hosting and updates.

## Summary

The video demonstrates a complete workflow for creating professional, animated product landing pages using AI tools and Claude Code. The process starts by generating product animation videos with Nano Banana 2 and Cling 3.0 through the key.ai platform.

### Video Creation Process

You create two product images (start and end frames) using Nano Banana 2, then animate between them using Cling 3.0. Claude helps craft the animation prompt by describing the desired transition. The resulting video becomes the core visual element for the website.

### Website Generation with Claude Code

Using Claude Code in Visual Studio Code with two custom skills (video-to-website and front-end design), you simply provide the video and basic requirements. Claude Code analyzes the video, extracts frames using ffmpeg, and builds a complete website with:

* Scroll-driven animations where frames sync with scroll position

* Professionally written copy and branding (generated automatically) * Modern design with smooth animations and dark mode

* Complete HTML, CSS, and JavaScript implementation

The system works in **plan mode** first, allowing you to review and adjust before implementation. You can provide feedback (like adjusting animation timing) and Claude Code will update the code accordingly.

### Deployment Pipeline

Once satisfied with the local version, you deploy using a professional pipeline: 1. **GitHub integration**: Claude Code pushes the codebase to a GitHub repository 2. **Vercel connection**: Import the repository to Vercel for automatic deployments 3. **Production updates**: Any code changes pushed to GitHub automatically sync to the live website

This creates separate testing (localhost), staging, and production environments. The video shows troubleshooting when frames weren't included in the initial GitHub push, demonstrating how to fix deployment issues.

### Business Applications

The creator suggests this approach enables offering **$5k-$10k website packages** to businesses by creating impressive demo sites in hours rather than months. You can specialize in niches (like blender companies), create custom demos, and offer ongoing maintenance for recurring revenue.
