* **Implement self-modifying prompts**: Use platform-specific `.md` files (e.g., `agents.md`, `claude.md`) to create systems that learn from mistakes and adapt over time.

## Summary

### Introduction and Core Concepts

Nick Saraev introduces a comprehensive course on AI agents, emphasizing that no programming experience is required. He runs a business generating over $4 million annually using AI agents and teaches over 2,000 people. The course begins with a demo showcasing five AI agents autonomously navigating Chrome browsers to perform economically valuable activities, such as filling out contact forms for lead generation.

He explains the **core agent loop**, which consists of three steps: **observation** (reading context, files, tool calls), **reasoning** (planning based on goals), and **action** (executing tool calls). This loop repeats until a **definition of done** is met, a critical component often missing in prompts. Saraev stresses that AI agents are more than just large language models; they combine LLMs with tools, memory, and orchestration frameworks.

### Platform Setup and Comparisons

The course covers three major AI agent platforms: **Codeex** (OpenAI), **Claude Code** (Anthropic), and **Anti-gravity** (Google). Saraev demonstrates signing up and basic usage for each, showing how to create a simple portfolio website. He compares their strengths:

* **Claude Code** excels in interpretable reasoning and orchestration.

* **Anti-gravity** (Gemini) is superior for front-end design and multimodal tasks.

* **Codeex** (GPT) is best for back-end programming and test-driven development.

While differences are minor, choosing the right model for specific tasks can yield marginal improvements.

### Advanced Prompting and Skill Development

Saraev delves into **self-modifying agent instructions** using `.md` files. By appending rules to these files after mistakes or feedback, agents accumulate knowledge, reducing errors over sessions. This pattern applies globally (`claude.md`) and locally (`project.md`).

He introduces **agent skills**—repeatable workflows that standardize outputs. Skills are files with YAML front matter (name, description) and detailed instructions. Saraev demonstrates creating algorithmic art using a skill from Anthropic's repository, highlighting how skills turn statistical variance into deterministic paths.

### Multi-Agent Orchestration Patterns

The course explores several advanced orchestration techniques:

* **Multi-agent MCP orchestration**: Using Claude as a manager to delegate tasks to Gemini (front-end) and Codeex (back-end), then validating results.

* **Video-to-action pipelines**: Agents learn from YouTube tutorials by using Gemini's video understanding API to extract step-by-step instructions, then execute them via tools.

* **Stochastic multi-agent consensus**: Spawning multiple agents with slight prompt variations to traverse the search space of ideas, aggregating results for better outcomes.

* **Agent chat rooms**: Creating centralized spaces where agents with different personalities debate ideas, leading to higher-quality insights.

* **Sub-agent verification loops**: Having fresh-context agents review the output of biased implementer agents to catch errors objectively.

### Prompt Contracts and Reverse Prompting

To combat vague tasks, Saraev introduces **prompt contracts**, where agents draft structured agreements with goals, constraints, formats, and failure conditions before proceeding. This formalizes scope and improves consistency.

**Reverse prompting** extends this by having agents ask clarifying questions upfront, ensuring one-shot success. Saraev shows building a website using these techniques, resulting in higher-quality outputs.

### Practical Applications and Scaling

Saraev revisits the initial demo, explaining the **multi-agent Chrome MCP manager**. This pattern spawns parallel Chrome instances for tasks like form filling or web scraping, dramatically increasing throughput. He demonstrates finding rental apartments using four simultaneous agents.

He discusses **context management**, detailing how tokens are consumed by system prompts, memory files, skills, and conversation history. The **iceberg technique** optimizes context by storing only essential data in the prompt while providing tools to access additional information on demand.

### Cost Optimization and Model Selection

Saraev advises using a **60/30/10 rule** for model allocation: 60% of tokens on cheap models (e.g., Haiku) for simple tasks, 30% on mid-tier models (e.g., Sonnet), and 10% on top models (e.g., Opus) for routing and critical decisions. This balances cost and quality, especially for large-scale operations like lead scraping.

He provides a cheat sheet on LLM API pricing and mentions batch APIs for cost-efficient bulk processing.

### Conclusion

Saraev emphasizes that AI agents represent a significant economic shift and encourages viewers to master these techniques. He offers resources in the video description and invites engagement for future content.
