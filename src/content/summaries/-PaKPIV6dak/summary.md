## Key Takeaways

This video reveals a significant disconnect between developer perception and reality regarding AI coding tool productivity. • Developers using **AI tools were 19% slower** than those working without them • Developers **predicted 24% faster** performance but still believed they were 20% faster after experiencing slowdowns • The **43 percentage point gap** between perception and reality highlights cognitive bias • **84% of developers** now use AI coding tools, but only 52% report positive productivity impact

## Summary

A controlled study conducted by research group MER found that developers using AI coding tools were actually 19% slower than those working without any AI assistance. This contradicts the widespread perception that AI tools dramatically boost developer productivity.

The study involved 16 experienced open source developers working on 246 real issues in production code, not toy problems. Before using the tools, developers predicted they would be 24% faster with AI assistance. Even after experiencing the actual slowdown, they still believed they were 20% faster—creating a 43 percentage point gap between perception and reality.

### Broader Industry Context

The video presents additional concerning data points about AI tool adoption and impact. While 84% of developers now use AI coding tools, only 52% report positive productivity impact. Code review times have increased by 91%, suggesting AI-generated code requires more scrutiny. Perhaps most alarmingly, AI-assisted code bases show more security vulnerabilities.

### The Productivity Paradox

One developer's comment highlights the central question: "If productivity is up 1,000x now, where is all the new revolutionary software?" The video references the most liked comment on Carpety's interview (120 likes) which states, "The revolution is already here. Cool. What did you build? Sleepy time turns the lights off." This suggests that despite widespread AI adoption, there hasn't been a corresponding explosion in groundbreaking software innovation.

The video concludes by noting that "the human cost is even harder to ignore," setting up for part five of the series which will presumably explore the broader implications of these findings.

## Context

This research matters because AI coding tools have become ubiquitous in software development, with 84% adoption rates among developers. Understanding the actual versus perceived impact of these tools is crucial for teams making technology decisions and for developers managing their workflows. The findings challenge the narrative that AI tools universally boost productivity and suggest organizations need to implement AI assistance more thoughtfully, considering both the potential slowdowns and increased security risks. This connects to broader trends in developer tooling and the ongoing debate about AI's role in creative and technical work.
