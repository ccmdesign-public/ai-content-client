# Large YAML Data Files Committed to Repository

## Problem Statement

The PR adds 98 per-tag YAML files plus an `index.yml` and a `tags-index.json` to the repository. The `ai-general.yml` alone is 1,645 lines. These are generated/synced data files that add ~18,000+ lines of data content to the repo. Over time, as tags grow, this will increase repository size and Git history.

## Findings

- **Location**: `src/content/tags/*.yml`, `src/content/tags-index.json`
- **Evidence**: PR adds 18,679 lines, 111 changed files. The vast majority are data files.
- **Impact**: Repository clone/checkout size increases. Git diffs on these files are noisy. These files are regenerated by `npm run sync:tags` and could be treated as build artifacts.
- **Current State**: The `tags-index.json` is imported statically by `useTagsConfig.ts`, and the YAML files are consumed by Nuxt Content as a data collection.

## Proposed Solutions

### Solution A: Accept as-is but add to .gitattributes (Recommended)
- Mark YAML tag files as generated in `.gitattributes` so they collapse in PR diffs.
- **Pros**: Simple. Files still tracked for deployment.
- **Cons**: Still increases repo size.
- **Effort**: Small
- **Risk**: Low

### Solution B: Generate at build time instead of committing
- Run `sync:tags` as a build step and `.gitignore` the generated files.
- **Pros**: Cleaner repo. No data files in version control.
- **Cons**: Requires scraper repo availability at build time. Breaks Netlify static builds unless handled.
- **Effort**: Medium
- **Risk**: Medium
