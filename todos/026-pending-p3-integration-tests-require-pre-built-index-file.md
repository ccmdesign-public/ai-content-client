# Integration Tests Require Pre-Built Index File

## Problem Statement

The integration tests in `buildSearchIndex.test.ts` read from `src/public/search-index.json` which must be generated by running `npm run build:search-index` before tests. If the file doesn't exist, all 7 tests fail. There's no CI step shown that ensures the index is built before `vitest run`.

## Findings

- **Location**: `src/tests/scripts/buildSearchIndex.test.ts`
- **Evidence**: `const INDEX_PATH = resolve(...)` pointing to the output file, with `it('search-index.json exists', ...)` as the first test.
- **Impact**: Tests will fail in CI unless the build step is explicitly configured. The test `indexes at least 300 documents` is also brittle -- it will fail if the content count drops below 300.

## Proposed Solutions

### Solution A: Run build:search-index in CI before tests
- Add a CI step or a vitest globalSetup that runs the index build.
- **Effort**: Small
- **Risk**: Low

### Solution B: Make the test self-contained
- Have the test build a temporary index from fixture data rather than depending on the full content directory.
- **Effort**: Medium
- **Risk**: Low
